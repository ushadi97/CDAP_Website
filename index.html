ABOUT CSI

Selecting the most suitable candidates for interviews is an important process for organizations that can affect their overall work performance. Typically, recruiters check Curriculum Vitae (CV), shortlist them and call candidates for interviews which have been the way of recruiting new employees for a long time. To minimize the time spent on the above process, pre-screening mechanisms are nowadays implemented by organizations. However, those mechanisms need sufficient information to evaluate the candidate . For example, in case of a software engineer, the recruiters are interested on the programming ability, academic performance as well as personality traits of potential candidates. In this research, a pre-screening solution is proposed to screen the applicants for the post of Software Engineer where candidates are screen based on an initial call transcript, GitHub profile, LinkedIn profile , CV, Academic transcript and, Recommendation letters. This approach extracts textual features of different dimensions based on Natural Language Processing to identify the Big Five personality traits, CV and GitHub insights, candidate’s skills, background, and capabilities from Recommendation letters as well as programming skills and knowledge from Academic transcript and Linked Profile. The results obtained from the different areas are presented and shown that the selected supervised machine learning algorithms and techniques can be used to evaluate the best possible candidates.

RESEARCH PROBLEMS

Traditional recruitment methods are not effective when comes to selecting a right candidate for Software Engineer positions.


More human and capital resources are wasted when comes to recruiting a candidate


Recruitment management systems that are currently available do not provide better results due to less analyzing factors


It requires a considerable amount of time and effort for recruiters to decide and choose a candidate and shortlist for the interview

RESEARCH OBJECTIVES

Introducing an optimal solution which helps recruiters for selecting the most suitable candidates before the initial screening of the interview for the position of Software Engineer using less resources and time thus makes recruitment process accurate and efficient. Also provides support to candidates where they can evaluate their Skills.

Reduce the human and capital resources by avoiding the traditional interviewing process

Moving to an advanced platform of machine learning techniques by introducing major dimensions to evaluate a candidate for Software Engineer

The system can evaluate the ideal candidate based on an initial phone call transcript, GitHub profile, LinkedIn profile, CV, and Recommendation letters

METHODOLOGY

The proposed system determines the candidate’s statistics by analyzing their user profiles using different techniques and methodologies. Machine learning and NLP techniques are carried out. The proposed work consists of 4 major components which is given below.

Big five personality Analysis

Amazon transcribed service is used to analyze the recorded audio call to a text. Big five personality model is used to predict the personality traits. Logistic Regression model used to test future predictions

CV & GitHub Analysis

“PDFMiner” python library is used to extract the text from the pdf document, CV. NLTK steps are used. Naïve-Bayes model is used


An API request to GitHub is made, the request is responded with JSON type meta information

Recommendation letter & Questionnaire Analysis

Academic, work, social, and personal competencies identified from the recommendation letters & label as highly recommend, recommend, neutral & weakly recommend. Support Vector Machine model is used


MCQ questionnaires for each programming languages to identify programming skills of the candidate

LinkedIn & Academic transcript

The automated script created using selenium to login to the LinkedIn profiles. Neural network model used to categorize available skills. Compares available skills of the candidate are matching with required skills.


Breakdown to semesters, module names and their respective grades. Tesseract.js OCR to separate, recognize the module names. Naïve Bayes model is used
